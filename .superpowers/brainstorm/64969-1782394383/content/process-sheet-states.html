工艺单状态流转 & 业务约束

保存 → 确认 → 下发，严格校验，软删除

工艺单状态机

📄 保存
草稿状态，可编辑
→
✅ 确认
已定稿，不可编辑
→
🚀 下发
发至车间，
此时锁定全部关联数据
🔄 状态操作规则
📄 保存 → 确认：自由操作
✅ 确认 → 下发：
⚠ 必须校验合同版本号是否为最新！
如合同有新版本，禁止下发，需重新生成工艺单
🚀 已下发：工艺单只读，不可修改
↩ 确认 → 保存：可撤回修改
↩ 下发不可回退
🗑️ 删除规则（软删除）
所有表都使用 is_deleted 字段
实际数据保留，查询时过滤
合同删除限制：
✅ 无工艺单 → 可删除（软删除）
✅ 工艺单保存/确认 → 可删除（软删除，工艺单同步标记）
❌ 工艺单已下发 → 不允许删除合同
工艺单删除限制：
保存/确认中 → 可删除（软删除）
已下发 → 不可删除
🔍 下发工艺单时的校验流程
1. 获取工艺单关联的合同 contract_id
2. 获取合同的 最新确认版本号 (max confirmed version)
3. IF 工艺单的 confirm_version_no < 合同最新版本号
   → 禁止下发，提示："合同已有新版本，请重新生成工艺单"
4. IF 工艺单的 confirm_version_no == 合同最新版本号
   → 允许下发，状态变更为 已下发
5. 下发后 = 所有关联数据锁定（合同不可删、工艺单不可改）

# 全流程生产管理 — 设计文档

> 版本：2.0
> 日期：2026-06-28
> 状态：待评审

## 概述

为嘉元瑞通工厂管理系统添加完整的生产全流程管理，涵盖从合同确认到成品完成的全部工序追踪、状态变更通知、外部协作人员管理。

## 背景

当前系统合同状态仅三段式：草稿 → 保存 → 已下发。实际生产需要按工序逐级推进，且生产分为两条并行线：坯布计划线和工艺单线。

### 核心变更（V2）

| 项目 | V1（旧） | V2（新） |
|------|---------|---------|
| 合同确认方式 | 客户通过公开链接确认 | 销售经理通过企业微信群确认 |
| 客户确认 | 合同层面 | 工艺单层面（公开链接） |
| 下推工艺单 | 整个合同下推 | 行项目级别独立下推 |
| 生产推进 | 单线串行 | 双线并行（坯布计划 + 工艺单）|
| 版本号 | 确认图有版本号 | 无版本号，仅记录历史 |
| 合同状态 | 草稿→保存→已下发 | 草稿→确认→已下发 |

### 生产双线并行架构

```
合同（草稿）
  │ 销售经理企业微信群确认
  ▼
合同（确认）
  │
  ├── 线A: 下达坯布计划 ──→ 织造中 ──→ 织造完成
  │         （外协人员负责）
  │
  └── 线B: 下推行项目工艺单 ──→ 定型中 ──→ 刷毛烫光中 ──→ 印花中 ──→ 成品缝制 ──→ 成品完成
            （工艺单流转各车间，逐级推进汇报）
  
 两条线可同步进行，通常先下达坯布计划再下发工艺单。
 最终通过合同串联合并所有工序进度。
```

## 一、角色与权限

### 角色矩阵

| 角色 | 原角色 | 新增能力 | 页面访问 |
|------|--------|---------|---------|
| 业务员 | 不变 | 取消自己的合同（录入已产数量） | 全部 + 状态日志查看 |
| 销售经理 | 不变 | 主管确认、取消合同、回退/返工 | 全部 |
| 生产专员 | 不变 | 管理工序主数据、配置工序负责人、回退/返工 | 全部 + 工序管理页 |
| **外协人员（新增）** | — | 仅查看被分配的合同行项目，推进织造完成 | 专属任务清单页 |

### 外协人员限定

- 只能看到自己被指定为「坯布负责人」的合同行项目
- 权限：仅操作「推进到织造完成」
- 不可查看合同金额、客户信息、其他行项目

## 三、状态机设计

### 3.1 合同状态流转

```
草稿 ──（销售经理企业微信群确认）──→ 确认 ──（行项目下推/坯布下达）──→ 已下发
         └──（管理员手动确认，带确认意见）
```

- **草稿**: 可编辑、可发企业微信群让销售经理确认
- **确认**: 不可编辑，行项目可开始下推工艺单或下达坯布计划
- **已下发**: 至少一个行项目已进入生产，合同完成流转

### 3.2 行项目生产双线并行

行项目确认后，同时或先后开启两条生产线：

```
                         行项目确认
                            │
              ┌─────────────┴─────────────┐
              ▼                            ▼
    ┌──────────────────┐      ┌──────────────────────────┐
    │ 线A: 坯布计划线   │      │ 线B: 工艺单线             │
    ├──────────────────┤      ├──────────────────────────┤
    │ ① 下达坯布计划    │      │ ① 下推行项目为工艺单      │
    │ ② 织造中          │      │ ② 定型中 → 定型完成      │
    │ ③ 织造完成        │      │ ③ 刷毛烫光中→刷毛烫光完成 │
    │                   │      │ ④ 印花中 → 印花完成      │
    │                   │      │ ⑤ 成品缝制 → 成品完成    │
    └──────────────────┘      └──────────────────────────┘

     可同步进行 / 先后顺序不限        工艺单流转各车间汇报
```

**关键规则：**
- 线A（坯布计划）由外协人员执行，推进到织造完成
- 线B（工艺单）由各车间内部人员逐级推进，从定型开始到成品完成
- 两条线互不阻塞，可独立推进
- 行项目必须「确认」后才能下推工艺单或下达坯布计划

### 3.3 操作类型

| 操作 | 说明 | 谁可以操作 |
|------|------|-----------|
| 推进（advance） | 按顺序走到下一工序 | 下一工序负责人 |
| 回退（rollback） | 退回上一道工序（纠错） | 生产专员/销售经理 |
| 返工（rework） | 退回指定工序（批量问题） | 生产专员/销售经理 |
| 取消（cancel） | 行项目终止 | 业务员 / 销售经理 / 生产专员 |
| 坯布下达（yarn_plan_release） | 下达坯布计划并指定负责人 | 生产专员/销售经理 |
| 合同确认（confirm） | 销售经理确认合同 | 销售经理 |
| 手动确认（manual_confirm） | 管理员紧急确认，需带确认意见 | 管理员（admin） |

### 3.4 各工序通知规则

| 工序 | 通知对象 | 说明 |
|------|---------|------|
| 合同待确认 | 企业微信群 @销售经理 | 业务员提交确认请求 |
| 坯布计划下达 | contract_item.yarn_plan_user_id | 外协织造人员 |
| → 推进到织造完成 | contract_item.yarn_plan_user_id | 外协人员只管自己这部分 |
| → 定型及以后各工序 | 该工序全部配置负责人 | 内部人员全员通知 |
| 回退/返工 | 涉及工序的全体负责人 | |
| 取消 | 业务员 + 全部工序负责人 | |
| 成品完成 | 业务员 + 销售经理 | |

## 四、界面交互

### 4.1 合同详情页 — 行项目进度

合同详情页行项目表格增加「生产进度」列，行项目操作按钮区分两条线：

**线A操作（坯布计划线）：**
- 「下达坯布」按钮 — 未开始的行项目显示，指定外协人员
- 外协人员看到该行项目后推进：坯布计划→织造中→织造完成

**线B操作（工艺单线）：**
- 「下推工艺单」按钮 — 确认后可点击，生成关联工艺单
- 各车间在工艺单上逐级推进

**按钮显示规则：**

| 状态 | 线A按钮 | 线B按钮 |
|------|---------|---------|
| 未确认 | 禁用 | 禁用 |
| 已确认，坯布未下达，工艺单未下推 | 下达坯布 | 下推工艺单 |
| 坯布已下达，织造中 | 推进 | 下推工艺单 |
| 工艺单已下推，定型中 | 完成 | 在工艺单推进 |
| 两条线都完成 | — | — |

## 五、企业微信通知实现

### 5.1 架构

```
系统状态变更 → Service 层调用 NotifyService 
  → POST 到企微群机器人 Webhook URL
  → 群内出现通知消息
```

无需域名，无需回调，纯出站 HTTP 请求。

### 5.2 消息格式

```
📋 生产进度通知
合同：C20260001-1 | 200*240/4KG/单层
当前：定型中（已推进）
操作人：张三
下一道：刷毛烫光 → 请安排
```

含有用户列表时可 @提及，格式参照企微机器人文档。

### 5.3 通知失败处理

- 通知失败不影响业务操作（异步发送）
- 在 production_log 中记录通知发送状态（success/failed）
- 提供手动重试按钮（管理页面）

## 六、合同级别状态映射

合同有两个层面的状态：
- **核心状态**（contract.status）：草稿 → 确认 → 已下发，记录合同生命周期
- **派生展示状态**（computed_status）：从行项目生产状态自动计算

### 6.1 核心状态

| 条件 | contract.status |
|------|----------------|
| 新建未确认 | 草稿 |
| 销售经理确认 | 确认 |
| 至少一个行项目已下推或下达坯布 | 已下发 |

### 6.2 派生展示状态（computed_status）

| 条件 | computed_status |
|------|----------------|
| 全部行项目为 NULL（未生产）且 status=草稿 | 草稿 |
| 全部行项目为 NULL（未生产）且 status=确认 | 确认 |
| 至少一行已有生产状态 | 已下发 |
| 全部行项目已完成 | 已完成 |
| 全部行项目已取消 | 已取消 |

## 七、工艺单客户确认

合同确认（销售经理企业微信群确认）之后，工艺单层面增加客户确认环节：

### 7.1 流程

```
合同行项目下推 → 创建工艺单（草稿）
  → 生成确认图 + 公开确认链接（UUID令牌）
  → 发送给客户（微信）
  → 客户打开链接，查看工艺单详情
  → 客户提交确认意见（或退回修改）
  → 工艺单变为「已确认」，记录客户意见
  → 工艺单可继续流转车间
```

### 7.2 关键规则

- 工艺单可反复生成确认图（每次生成快照，但不记录版本号）
- 生成公开链接（UUID令牌），客户无需登录
- 客户确认后工艺单变为「已确认」
- 客户可退回修改，退回后工艺单变为「修改中」
- 只有已确认的工艺单才能流转到车间工序
- 不记录版本号，但操作记录记入 production_log

## 八、安全与约束

1. 外协人员只能通过 JWT 认证访问，与内部用户同一认证体系
2. 权限校验在 Service 层实现，API 路由不直接暴露修改操作
3. 状态变更操作幂等判断：不允许推进一个已经推进过的工序
4. 行项目取消后不可恢复
5. 坯布计划下达后不可修改坯布负责人

# 全流程生产管理 — 设计文档

> 版本：1.0
> 日期：2026-06-28
> 状态：待评审

## 概述

为嘉元瑞通工厂管理系统添加完整的生产全流程管理，涵盖从合同确认到成品完成的全部工序追踪、状态变更通知、外部协作人员管理。

## 背景

当前系统合同状态仅三段式：草稿 → 保存 → 已下发。实际生产需要按工序逐级推进（坯布计划→织造→定型→刷毛烫光→印花→成品缝制→成品完成），并要求：
- 行项目（contract_item）级别独立追踪进度
- 每道工序可配置负责人，状态变化时通过企业微信群机器人通知
- 支持回退、返工、取消
- 坯布计划下达给外协织造厂人员，仅其可见专属任务

## 一、角色与权限

### 角色矩阵

| 角色 | 原角色 | 新增能力 | 页面访问 |
|------|--------|---------|---------|
| 业务员 | 不变 | 取消自己的合同（录入已产数量） | 全部 + 状态日志查看 |
| 销售经理 | 不变 | 主管确认、取消合同、回退/返工 | 全部 |
| 生产专员 | 不变 | 管理工序主数据、配置工序负责人、回退/返工 | 全部 + 工序管理页 |
| **外协人员（新增）** | — | 仅查看被分配的合同行项目，推进织造完成 | 专属任务清单页 |

### 外协人员限定

- 只能看到自己被指定为「坯布负责人」的合同行项目
- 权限：仅操作「推进到织造完成」
- 不可查看合同金额、客户信息、其他行项目

## 二、数据结构

### 2.1 工序主数据表（process_step）— 新增

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | |
| step_code | VARCHAR(30) UNIQUE | 工序编码，如 `weaving`, `setting` |
| step_name | VARCHAR(50) | 工序名称，如「织造」|
| sort_order | INT | 排序号，决定流转顺序 |
| is_active | BOOLEAN | 是否启用，默认 true |
| created_at | DATETIME | |

> 系统预置种子数据：坯布计划 → 织造 → 定型 → 刷毛烫光 → 印花 → 成品缝制 → 成品完成

### 2.2 工序负责人配置表（process_step_assignee）— 新增

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | |
| process_step_id | INT FK → process_step | 关联工序 |
| user_id | INT FK → user | 关联用户 |
| created_at | DATETIME | |

> 多对多关系：一个工序可有多个负责人。坯布计划工序特殊，负责人由下达时临时指定，不在此表配置。

### 2.3 状态日志表（production_log）— 新增

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | |
| contract_id | INT FK → contract | 关联合同 |
| contract_item_id | INT FK → contract_item, NULLABLE | 关联行项目（NULL 表示合同级操作）|
| from_status | VARCHAR(30), NULLABLE | 变更前状态（NULL 表示初始）|
| to_status | VARCHAR(30) | 变更后状态 |
| operation_type | ENUM('推进','回退','返工','取消','确认','坯布下达') | 操作类型 |
| operator_id | INT FK → user | 操作人 |
| remark | TEXT, NULLABLE | 备注 |
| created_at | DATETIME | 操作时间 |

索引：
- `(contract_item_id, created_at)` — 按行项目查日志
- `(contract_id, created_at)` — 按合同查日志

### 2.4 contract_item 表变更

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| production_status | VARCHAR(30), NULLABLE | 当前工序 step_code，NULL 表示未进入生产 |
| yarn_plan_user_id | INT FK → user, NULLABLE | 坯布负责人（外协人员），坯布计划下达时指定 |
| cancel_quantities | JSON, NULLABLE | 取消时各工序已产数量 `{"weaving":5, "setting":2}` |
| cancel_reason | TEXT, NULLABLE | 取消原因 |

### 2.5 user 表变更

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| wecom_userid | VARCHAR(100), NULLABLE | 企业微信用户 ID，用于通知 @提及 |
| role | ENUM 扩展 | 新增 `外协人员` 枚举值 |

### 2.6 系统配置表（system_config）— 新增

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | |
| config_key | VARCHAR(100) UNIQUE | 配置键 |
| config_value | TEXT | 配置值 |

预置配置项：
- `wecom_webhook_url` — 企业微信群机器人 Webhook 地址
- `production_notify_enabled` — 是否启用生产通知

## 三、状态机设计

### 3.1 行项目状态流转

```
                    ┌──────────────────────────┐
                    │   NULL（待下达坯布计划）  │
                    └──────────┬───────────────┘
                               │ 坯布计划下达（指定坯布负责人）
                               ▼
                    ┌──────────────────────────┐
                    │   坯布计划已下达           │
                    │   (yarn_plan_released)     │
                    └──────────┬───────────────┘
                               │ 外协人员推进
                               ▼
                    ┌──────────────────────────┐
                    │   织造中 (weaving)        │
                    └──────────┬───────────────┘
                               │ 外协人员推进
                               ▼
                    ┌──────────────────────────┐
                    │   织造完成                │
                    └──────────┬───────────────┘
                               │ 定型负责人推进
                               ▼
                    ┌──────────────────────────┐
                    │   定型中 → 定型完成       │
                    └──────────┬───────────────┘
                               │ 刷毛负责人推进
                               ▼
                    ┌──────────────────────────┐
                    │   刷毛烫光中 → 刷毛烫光完成│
                    └──────────┬───────────────┘
                               │ 印花负责人推进
                               ▼
                    ┌──────────────────────────┐
                    │   印花中 → 印花完成       │
                    └──────────┬───────────────┘
                               │ 成品负责人推进
                               ▼
                    ┌──────────────────────────┐
                    │   成品缝制 → 成品完成     │
                    └──────────────────────────┘

   任意状态 ──→ 已取消 (cancelled) [带取消原因和已产数量]
   任意状态通过回退/返工 → 上一道或指定工序
```

### 3.2 操作类型

| 操作 | 说明 | 谁可以操作 |
|------|------|-----------|
| 推进（advance） | 按顺序走到下一工序 | 下一工序负责人 |
| 回退（rollback） | 退回上一道工序（纠错） | 生产专员/销售经理 |
| 返工（rework） | 退回指定工序（批量问题） | 生产专员/销售经理 |
| 取消（cancel） | 行项目终止 | 业务员（仅自己合同）/ 销售经理 |
| 坯布下达（yarn_plan_release） | 下达坯布计划并指定负责人 | 生产专员/销售经理 |

### 3.3 各工序通知规则

| 工序 | 通知对象 | 说明 |
|------|---------|------|
| 坯布计划下达 | contractor_item.yarn_plan_user_id | 外协织造人员 |
| → 推进到织造完成 | contractor_item.yarn_plan_user_id | 外协人员只管自己这部分 |
| → 定型及以后各工序 | 该工序全部配置负责人 | 内部人员全员通知 |
| 回退/返工 | 涉及工序的全体负责人 | |
| 取消 | 业务员 + 全部工序负责人 | |
| 成品完成 | 业务员 + 销售经理 | |

## 五、企业微信通知实现

### 5.1 架构

```
系统状态变更 → Service 层调用 NotifyService 
  → POST 到企微群机器人 Webhook URL
  → 群内出现通知消息
```

无需域名，无需回调，纯出站 HTTP 请求。

### 5.2 消息格式

```
📋 生产进度通知
合同：C20260001-1 | 200*240/4KG/单层
当前：定型中（已推进）
操作人：张三
下一道：刷毛烫光 → 请安排
```

含有用户列表时可 @提及，格式参照企微机器人文档。

### 5.3 通知失败处理

- 通知失败不影响业务操作（异步发送）
- 在 production_log 中记录通知发送状态（success/failed）
- 提供手动重试按钮（管理页面）

## 六、合同级别状态映射

| 条件 | 合同状态 |
|------|---------|
| 全部行项目为 NULL（未坯布下达） | 草稿 |
| 主管已确认 | 确认 |
| 至少一行已坯布下达 | 坯布计划已下达 |
| 全部行项目已完成 | 已完成 |
| 有行项目已取消 | 部分取消 |
| 全部行项目已取消 | 已取消 |

## 七、安全与约束

1. 外协人员只能通过 JWT 认证访问，与内部用户同一认证体系
2. 权限校验在 Service 层实现，API 路由不直接暴露修改操作
3. 状态变更操作幂等判断：不允许推进一个已经推进过的工序
4. 行项目取消后不可恢复
5. 坯布计划下达后不可修改坯布负责人
